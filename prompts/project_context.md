# 项目上下文

## 现状

- 旧仓库已经具备多模型图像标注、flowchart mermaid 融合、印章文字抽取、accepted/review/failed 判定等能力。
- 当前真实任务没有变，仍然以图表、流程图、印章识别为主。
- 变化点不在任务类型，而在上游和交付格式：
  - 上游要切到本地部署服务
  - 当前已知上游是 `MinerUPro 2.5` 与 `qwen 122b`
  - 最终输出要尽量统一到 MinerU

## 目标

- 建一个独立仓库，不和旧仓库嵌套
- 保留旧仓库里跟图表、流程图、印章识别强相关的成熟能力
- 不继续沿用旧仓库的 `final_label` 作为最终交付格式
- 最终统一写出 MinerU 风格 `content_list_v2.json` / `content_list.json`

## 当前架构假设

- `MinerUPro 2.5` 作为结构锚点：
  - 页面结构
  - 块切分
  - bbox
  - 阅读顺序
- `qwen 122b` 作为语义仲裁器：
  - 图表/流程图说明
  - 印章文字修正
  - caption 补全
  - 结构化内容补强

## 仲裁策略

- 当前只有两路上游，因此不做“多数投票”
- 采用 `MinerU 主锚点 + Qwen 补充 + review 回退`
- 需要保留来源追踪和冲突原因
